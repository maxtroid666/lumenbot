# Бот с Claude на борту

Один бот, живёт в нескольких группах одновременно, в каждой группе своя отдельная память (история сообщений хранится по chat_id - идентификатору чата).

## Как бот решает, отвечать или нет

1. Ответили реплаем на его сообщение - отвечает.
2. Упомянули через @username - отвечает.
3. В сообщении есть триггер-слово (список в `TRIGGER_WORDS`) - отвечает.
4. Иначе - с небольшим шансом (`RANDOM_REPLY_CHANCE`) может влезть сам.

Все сообщения чата запоминаются в любом случае - это и есть память для контекста.

## Шаг 1. Создать бота в Telegram

1. Написать **@BotFather** в Telegram, команда `/newbot`, задать имя и username.
2. Получить токен вида `123456:ABC-DEF...` - это `BOT_TOKEN`.
3. Обязательно выполнить `/setprivacy` -> выбрать бота -> **Disable**.
   Без этого бот в группах видит только сообщения, адресованные ему напрямую, и не сможет читать общий чат для памяти и триггеров.

## Шаг 2. Получить ключ Anthropic API

В консоли Anthropic (console.anthropic.com) создать API-ключ - это `ANTHROPIC_API_KEY`.

## Шаг 3. Деплой на Railway

1. Залить эту папку в GitHub-репозиторий (или сразу подключить папку в Railway).
2. В Railway: New Project -> Deploy from GitHub repo.
3. В Variables (переменные окружения) задать:
   - `BOT_TOKEN`
   - `ANTHROPIC_API_KEY`
   - остальные - по желанию, дефолты уже разумные (см. `.env.example`)
4. **Важно про память**: файловая система Railway по умолчанию временная (эфемерная) - при каждом передеплое файл базы данных `bot_memory.db` обнулится. Чтобы память сохранялась постоянно, нужно добавить Railway Volume (постоянный диск) и примонтировать его в путь, указанный в `DB_PATH`. Это делается в настройках сервиса -> Volumes -> Add Volume.

## Шаг 4. Добавить бота в чат

Просто пригласить бота в группу как обычного участника. Он сразу начнёт запоминать переписку и реагировать по правилам выше.

## Как поменять личность бота

Открыть `personality.txt`, переписать текст - это системный промт (system prompt), который определяет характер. Изменения подхватятся после следующего деплоя.

## Как раскинуть одного бота на несколько чатов

Ничего дополнительно делать не нужно - просто добавь того же бота в другую группу. Память по каждому чату ведётся отдельно (привязана к chat_id), личность и код общие.
